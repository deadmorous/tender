# Names: TensorObject names and index names

## Motivation

Formatted output relies on juxtaposition for tensor product.  To make
juxtaposition readable, every `TensorObject` name must render as a
single typographic symbol — a letter or a Greek/operator symbol from a
LaTeX command.  Index names follow mostly the same rule but additionally
allow short words used as descriptive labels (e.g. `vol`, `surf`).

Both kinds of name are value types in the C++ core.

---

## C++ name type

Names are short, immutable strings.  The infrastructure from `mpk_mix`
gives us a fixed-length string with strong-type wrapping and a
string-view companion at no runtime overhead:

```cpp
using Str16 = mpk::mix::FixedLengthString<16>;
MPKMIX_STRONG_STRING_VIEW(NameView);
MPKMIX_STRONG_TYPE(Name, Str16, mpk::mix::StrongStringFeatures<NameView>);
```

`FixedLengthString<16>` stores up to 16 characters inline (no heap
allocation).  16 is chosen because:

- The longest standard single-symbol LaTeX command is `\varepsilon`
  (10 chars).
- Descriptive index words like `boundary` are 8 chars.
- Names are stored *raw* (the renderer adds `\mathrm{…}` etc.); no
  LaTeX wrapper is ever stored in the name itself.
- 8 chars would be tight (`\varepsilon` alone exceeds it).

`TensorName` and `IndexName` are distinct strong types wrapping `Name`
so that the type system prevents accidental interchange.  They share the
same underlying storage and string features.

---

## TensorObject names

A `TensorObject` name must be a **single typographic symbol**:

| Form                  | Examples                        | Rule                            |
|-----------------------|---------------------------------|---------------------------------|
| Single ASCII letter   | `A`, `u`, `σ` (if UTF-8)        | exactly one character           |
| LaTeX command         | `\alpha`, `\nabla`, `\sigma`    | starts with `\`                 |

Multi-letter words (`sigma`, `nabla` spelled out) are **rejected** at
construction time.  The reason: juxtaposition of two-letter names is
unreadable (`AB` looks like a two-letter name, not `A` times `B`).

LaTeX rendering: the name is inserted as-is inside a math-bold or
upright-bold macro determined by the tensor's rank/display style.
For example, `\sigma` renders as `\boldsymbol{\sigma}` for a second-rank
tensor field.

---

## Index names

An index name follows the same single-symbol rule but additionally
**allows multi-letter ASCII words** for descriptive subscripts:

| Form                  | Examples                        | Rendered in LaTeX as            |
|-----------------------|---------------------------------|---------------------------------|
| Single ASCII letter   | `i`, `j`, `k`                   | `i`, `j`, `k` (italic)          |
| LaTeX command         | `\alpha`, `\mu`                 | `\alpha`, `\mu` (italic)        |
| Multi-letter word     | `vol`, `surf`, `node`, `bdry`   | `\mathrm{vol}` etc. (upright)   |

**Rendering rule (automatic):** if the stored name starts with `\` or
is exactly one character, render as-is (italic math mode default); if it
is two or more ASCII letters, wrap in `\mathrm{…}`.

Multi-letter index words must be short enough to be readable as
subscripts.  There is no hard character limit below 16, but names longer
than ~6 characters should be avoided in practice.

---

## Summary

| | `TensorName` | `IndexName` |
|---|---|---|
| Single letter | ✓ | ✓ |
| LaTeX command (`\…`) | ✓ | ✓ |
| Multi-letter word | ✗ | ✓ |
| Underlying storage | `Name` / `Str16` | `Name` / `Str16` |
| Max stored length | 16 chars | 16 chars |
| LaTeX wrapping stored? | no | no |
