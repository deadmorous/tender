# Phase 13.92 — Identity and theorem organisation

_Date: 2026-06-12_

## Motivation

The library has grown incrementally and the split between "identity" and
"theorem" has drifted: some provable statements live in the identity layer,
some rewrite rules are defined locally in examples, and the dependency
structure is undocumented.  Phase 13.92 establishes a clean, principled
organisation without touching core C++ or Python infrastructure.

---

## Principle

| Layer | Contains | Used as |
|---|---|---|
| `lib/identities/` | First-principle rewrite rules — axiomatic or promoted from proven theorems | `Identity` objects with `PatternVar`s; passed to `apply_identity`, `search_apply`, etc. |
| `lib/theorems.py` | Formal proofs that identity-layer statements hold | `Theorem` objects validated at import; **not** directly usable as rewrite steps |
| `examples/` | Demonstrations; import from both layers | Never define their own identities |

**Key constraint** — `Theorem` does not implement the `Identity` interface
(`apply_identity`, `find_matches`, `search_apply`).  To use a proved result
as a derivation step, an `Identity` counterpart with `PatternVar`s must exist
in `lib/identities`.  A theorem *validates* that identity; it does not replace
it.

---

## Current problems

1. **`eps_delta` appears as both an identity (axiom) and a theorem** —
   `definitions.py` lists it in `ALL` as a primitive; `theorems.py` proves it
   from first principles.  The identity is needed as a rewrite rule in the
   `bac_cab` proof, so it must stay in `definitions.py`, but it should be
   annotated as "proved, not axiomatic."

2. **`bac_cab` has no identity in the library** — after promotion to a theorem
   the rewrite rule was removed from `epsilon.py` (now `ALL = []`).  Both
   `bac_cab.py` and `bac_cab_search.py` define a local `Identity("BAC-CAB",
   ...)` instead of importing from `lib/identities`.

3. **`identity_dot_vec` duplicated** — present in both `definitions.py`
   (`identity_vec`) and `identity_tensor.py` (`identity_dot_vec`).

4. **`vectors.py` mixes proved and unproved** — `dot_comm` and `cross_anticomm`
   are proved in `theorems.py`; `double_cross`, `jacobi`, and `cross_self_zero`
   are asserted without proofs.

5. **`functions.py` has a broken identity** — `sin_sq_plus_cos_sq` sets
   `lhs == rhs` (it is a no-op rewrite).

6. **`epsilon.py` is empty** — after promotion of `bac_cab` and
   `cross_anticommutativity` to theorems, the module exports `ALL = []` with
   no replacement.

## Proposed theorem set

### Already proved

| Name | Statement | Method |
|---|---|---|
| `eps_delta_theorem` | ε-δ identity | `by_components` |
| `dot_commutativity` | `a·b = b·a` | `by_components` |
| `cross_anticommutativity` | `a×b = -(b×a)` | `by_derivation` (cross_def, eps_anticomm, cross_def_rev) |
| `bac_cab` | `a×(b×c) = b(a·c)−c(a·b)` | `by_derivation` (cross_def ×2, eps_delta) |

### Phase 13.92 additions

All three are feasible without core changes:

**`cross_self_zero` — `a×a = 0`**
- Method: `by_components` — expand `ε:(a⊗a)` in WCS; all 27 terms cancel
  because ε is antisymmetric and a⊗a is symmetric.
- Dependency: none beyond `wcs`.

**`double_cross` — `(a×b)×c = b(a·c) − a(b·c)`**
- Method: `by_derivation` —
  1. `apply_identity_auto(cross_anticomm, lhs)` → `-(c×(a×b))`
  2. `apply_identity_auto(bac_cab_id, ...)` → `-(a(c·b) − b(c·a))`
  3. Algebraic simplification (Scale distributes) → `b(c·a) − a(c·b)`
  4. `_find_and_rewrite_all(dot_comm, ...)` × 2 → `b(a·c) − a(b·c)`
- Dependencies: `cross_anticomm` identity, `bac_cab` identity, `dot_comm` identity.
- **Feasibility note**: step 4 requires applying `dot_comm` inside nested
  `TensorProduct` nodes; depends on `_find_and_rewrite_all` reaching those
  sub-expressions.  May need to fall back to `by_components` if the
  structural match fails.

**`jacobi` — `a×(b×c) + b×(c×a) + c×(a×b) = 0`**
- Method: `by_components` (three simultaneous `bac_cab` expansions produce
  six tensor products that cancel pairwise; simpler than tracking rewrites).
- Dependencies: `wcs`.

### Future theorems (beyond Phase 13.92)

These require additional infrastructure and are listed for roadmap purposes only.

| Name | Statement | Blocker |
|---|---|---|
| `eps_ij_kl` | `ε_{ijk}ε_{isp} = δ_{js}δ_{kp}−δ_{jp}δ_{ks}` (abstract) | abstract dim tracking (δ_{jj}=3) |
| `eps_two_shared` | `Σ_{ij}ε_{ijk}ε_{ijp} = 2δ_{kp}` | same — no concrete analogue in abstract layer |
| `eps_full` | `ε_{ijk}ε_{ijk} = 6` | same |
| `cross_vec_dyad` | `cross(a, cross(b, I)) = b⊗a − (b·a)I` | see note below |
| `div_cross` | `∇·(a×b) = (∇×a)·b − a·(∇×b)` | differential operator calculus |
| `curl_cross` | `∇×(a×b) = b·∇a − a·∇b + a(∇·b) − b(∇·a)` | same |
| `vector_laplacian` | `∇×(∇×a) = ∇(∇·a) − Δa` | same |
| `sym_strain` | `ε = ½(∇u + (∇u)^T)` is symmetric | needs rank-2 symmetry predicate |
| `cauchy_stress_sym` | `σ = σ^T` from angular momentum | needs tensor symmetry + integration |

**Note on `cross_vec_dyad`** — `cross(a, cross(b, I)) = b⊗a − (b·a)I`

This identity is the BAC-CAB rule with `I` in the role of the third vector.  Mathematically:
```
a×(b×I) = ε:(a⊗(ε:(b⊗I))) = b⊗(a·I) − I(a·b) = b⊗a − (a·b)I
```
(using dot_comm: `a·b = b·a`).  Verified numerically for multiple (a,b) pairs.

**Correct tender code**: `cross(a, cross(b, I))` — NOT `a % I % b`.
In Python, `a % I % b` is left-associative: `(a×I)×b = -2(a⊗b)`, which is a different
(and trivial) identity.  The intended expression is `a % (b % I)` or equivalently
`cross(a, cross(b, I))`.

**Why it cannot be proved in Phase 13.92**: `cross_def` constrains both arguments
to rank-1, so `cross(b, I)` cannot be rewritten as `ddot(eps, tp(b, I))` via that
identity.  Similarly `eps_delta` constrains its third pattern variable to rank-1.
No existing step reduces a `CrossProduct` node whose second operand has rank > 1.

**What is needed**: either a rank-unconstrained `cross_def` (or a separate
`cross_def_rank2` with `_C.constrain_rank(2)`), plus the corresponding
`eps_delta_rank2: ddot(eps, tp(a, ddot(eps, tp(b, C)))) = tp(b, dot(a, C)) - tp(C, dot(a, b))`
where C is rank-2.  Both are Python-library changes (no C++ core required).

## Dependency graph (acyclic by construction)

```
axioms (definitions.py)
  ├── eps_anticomm ──────────────────────> cross_anticommutativity (theorem)
  ├── cross_def + eps_anticomm + cross_def_rev ──> cross_anticommutativity
  ├── eps_delta (identity) ──────────────> bac_cab (theorem)
  │     └── proved by: eps_delta_theorem (no further deps)
  └── cross_def ×2 + eps_delta ──────────> bac_cab (theorem)

vectors.py identities (rewrite rules, proved by theorems above)
  ├── cross_anticomm + bac_cab + dot_comm ──> double_cross (theorem)
  └── bac_cab ×3 + dot_comm ──────────────> jacobi (theorem)
```

No cycles.  Theorems only depend on identities at lower levels or on other
theorems already constructed earlier in the same module.
