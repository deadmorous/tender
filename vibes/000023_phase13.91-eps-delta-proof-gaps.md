# Phase 13.91 — Proving eps_delta from first principles

_Date: 2026-06-10_

## Context

`eps_delta` is currently an axiom in `definitions.py`:

```python
eps_delta = Identity(
    "eps-delta",
    lhs=ddot(eps, tp(_a, ddot(eps, tp(_b, _c)))),
    rhs=tp(_b, dot(_a, _c)) - tp(_c, dot(_a, _b)),
)
```

It encodes the coordinate-free consequence of the component-level identity:

```
ε_{ijk} ε_{jlm} = δ_{il} δ_{km} − δ_{im} δ_{kl}
```

The classical proof enumerates all (i,k,l,m) combinations in {1,2,3}, computes
`Σ_j ε_{ijk} ε_{jlm}` for each, and verifies it equals the δ-expression.
Our system currently lacks the machinery to do this.

## What is missing

### a) `LeviCivitaSymbol::sign()` / evaluate method

`LeviCivitaSymbol` (`src/include/tender/expr.hpp:578`) stores abstract index IDs
but has no method to compute +1/−1/0 from a concrete assignment of those IDs to
values in {1,2,3}.  Need:

```cpp
int LeviCivitaSymbol::sign(std::array<int,3> values);
// Returns +1, -1, or 0: even permutation of (1,2,3), odd, or repeated.
```

### b) No "sum over concrete dummy index" step

`contract_kronecker_step` eliminates δ contractions abstractly.  There is no
step that substitutes j ∈ {1,2,3} into an expression, evaluates it, and
accumulates the sum — which is exactly what computing `Σ_j ε_{ijk} ε_{jlm}`
requires.

### c) `_normalize_component_form` is blind to `LeviCivitaSymbol`

The function (`python/tender/__init__.py:516`) falls through to `expr.python()`
for ε nodes.  Two ε×ε products with different but contracted indices therefore
won't compare equal, making `prove_equal_by_components` fail even if the algebra
is correct.

### d) `prove_equal_by_components` handles only rank-0/1 normal forms

`eps_delta` produces rank-2 (dyadic) results — sums of `tp(b, dot(a,c))` terms.
The normalizer handles `AbstractIndexedSum` and scalar `Product` nodes but not
outer products of abstract components with distinct free indices.

## Proposed implementation path (Phase 13.91)

1. **`LeviCivitaSymbol::sign(values)`** — classify a concrete triple as even/odd
   permutation of (1,2,3) or containing a repeat → +1/−1/0.

2. **`evaluate_levi_civita_step(cs)`** — after `expand_levi_civita_step` has
   produced `ε_{ijk} e^i⊗e^j⊗e^k`, substitute the concrete index values and
   replace each `LeviCivitaSymbol` term by its rational coefficient (±1 or 0).
   Terms with coefficient 0 are dropped; the result is a `Sum` of at most six
   concrete basis-vector products.

3. **`sum_concrete_index_step(index_id)`** — takes a dummy abstract index ID and
   substitutes it with 1, 2, 3 in turn, evaluating each branch and accumulating
   the sum.  Works downstream of `evaluate_levi_civita_step`.

4. **`_normalize_component_form` extension** — handle products of rational ×
   basis-vector chains so that the component-form equality check works for
   rank-2 (and higher) invariant expressions.

5. **`Theorem.by_components` for rank-2 results** — verify that both sides of
   `eps_delta` reduce to the same concrete component sum after the above steps.

## Relation to other proofs

Once `eps_delta` is a proved theorem (not an axiom), the non-circular proofs of
`cross_anticommutativity` and `bac_cab` (Phase 13.9) become fully grounded:
every lemma in their derivation chains back to either `cross_def` (a definition)
or the component-level ε algebra.

## Classical proof strategy (from hand-written derivation)

A scanned hand-written derivation (`vibes/images/bac-cab.png`) provides a
clean, complete proof using the determinant formula.  The steps map directly
onto tender's step-based framework:

### Step outline

**1. Expand vectors in basis and apply cross-product definition twice:**
```
a × (b × c)
  = a_i e_i × (b_j e_j × c_k e_k)
  = a_i b_j c_k  e_i × (ε_{jks} e_s)
  = a_i b_j c_k  ε_{jks} ε_{isp}  e_p
```

**2. Express ε as a determinant of Kronecker deltas:**
```
ε_{ijk} = (e_i × e_j) · e_k = det([δ_{ia}]_{a=1,2,3})
```
i.e. the determinant whose columns are the rows δ_{i·}, δ_{j·}, δ_{k·}.

**3. Contract the pair ε_{jks} ε_{isp} using det(AB) = det A · det B:**

Cycle both ε's to bring the dummy index s to the first position:
```
ε_{jks} = ε_{sjk}  (cyclic, even permutation → same sign)
ε_{isp} = ε_{spi}  (cyclic, even permutation → same sign)
```
The product of the two determinants, sharing a row in s, equals det of the
matrix product — which contracts on s via δ_{sα}δ_{sβ} = δ_{αβ}:
```
ε_{sjk} ε_{spi} = det([δ_{ss} δ_{sp} δ_{si}; δ_{js} δ_{jp} δ_{ji}; δ_{ks} δ_{kp} δ_{ki}])
```

**4. Expand the 3×3 determinant by cofactors along the first row:**
```
= δ_{ss}(δ_{jp}δ_{ki} - δ_{ji}δ_{kp})
- δ_{js}(δ_{sp}δ_{ki} - δ_{si}δ_{kp})
+ δ_{ks}(δ_{sp}δ_{ji} - δ_{si}δ_{jp})
```

**5. Contract Kronecker deltas (δ_{ss}=3, δ_{js}δ_{sp}=δ_{jp}, etc.):**
```
= 3(δ_{jp}δ_{ki} - δ_{ji}δ_{kp})
  - (δ_{jp}δ_{ki} - δ_{ji}δ_{kp})
  + (δ_{kp}δ_{ji} - δ_{ki}δ_{jp})
= δ_{jp}δ_{ki} - δ_{ji}δ_{kp}
```

**6. Substitute back and contract free indices:**
```
a × (b × c) = a_i b_j c_k (δ_{jp}δ_{ki} - δ_{ji}δ_{kp}) e_p
            = a_i b_j c_i e_j  -  a_i b_i c_k e_k
            = (a·c) b  -  (a·b) c
```

### Implementation road map

This proof is more structured than brute-force enumeration because it gives
reusable determinant machinery:

- **`Det3` AST node** — represents a 3×3 determinant of Expr entries; can hold
  `KroneckerDelta` nodes as entries.
- **`eps_to_det_step`** — rewrites `LeviCivitaSymbol(i,j,k)` as `Det3` of the
  appropriate δ matrix (step 2 above).
- **`contract_eps_pair_step`** — detects a `Product(LeviCivitaSymbol, LeviCivitaSymbol)`
  sharing one dummy index, cycles both to bring dummy first, applies
  det(AB)=det A · det B, and returns a single `Det3` of contracted deltas
  (steps 3).  Requires noting the sign of each cyclic permutation.
- **`expand_det3_step`** — cofactor-expands a `Det3` along the first row,
  returning a Sum of Products of 2×2 sub-determinants (step 4).
- **Extended `contract_kronecker_step`** — must handle `δ_{ss}` → scalar 3
  in addition to its current off-diagonal contraction logic (step 5).
- **`_normalize_component_form` extension** — handle rank-2 (dyadic) expressions
  (step 6 produces tensor products of vectors).

The cyclic-permutation sign tracking in `contract_eps_pair_step` is the
subtlest piece: ε_{jks} = +ε_{sjk} (even cycle), which the step must assert
explicitly as a sub-step or inline comment.
