# Phase 13.9 — Non-circular proofs for cross-product theorems

_Date: 2026-06-10_

## Problem

The Phase 13.8 implementation of `tender.lib.theorems` contained circular
proofs:

- `cross_anticommutativity` was "proved" by applying the `anti_commutativity`
  identity from `epsilon.py` — but that identity *is* the statement `a×b =
  -(b×a)`, so the proof assumed its own conclusion.
- `bac_cab` was "proved" by applying the `bac_cab` identity from `epsilon.py`
  — same circular problem.

A theorem proved by asserting itself as an axiom adds no epistemic value.  The
fix: prove both results from simpler, independent primitives, and remove the
asserted identities.

## Design

### New primitive axioms added to `definitions.py`

Two new identities are added as axioms (assumed truths about the ε tensor):

```
eps_anticomm  ε:(a⊗b) = -ε:(b⊗a)     antisymmetry of the Levi-Civita tensor
eps_delta     ε:(a⊗(ε:(b⊗c))) = b(a·c) − c(a·b)   ε-δ contraction identity
```

These are *not* about cross products — they state properties of ε directly.
Both are added to `definitions.ALL`.

A helper identity `cross_def_rev` (ε:(a⊗b) = a×b) is defined but **not** added
to `ALL` because it is the reverse of `cross_def`, and having both in the
rewrite set would create an infinite cycle in `search_apply`.

### Proof of `cross_anticommutativity`

```
a×b
  = ε:(a⊗b)        [cross_def — definition of cross product]
  = -ε:(b⊗a)       [eps_anticomm — antisymmetry of ε]
  = -(b×a)          [cross_def_rev applied inside Scale(-1,…)]
```

Step 3 uses `_find_and_rewrite_all(cross_def_rev, after2)` because
`apply_identity_auto` only matches at the root, and after step 2 the
ε:(b⊗a) sub-expression sits inside a `Scale(-1, …)`.

### Proof of `bac_cab`

```
a×(b×c)
  = ε:(a⊗(b×c))          [cross_def at root]
  = ε:(a⊗(ε:(b⊗c)))      [cross_def in subtree, via _find_and_rewrite_all]
  = b(a·c) − c(a·b)       [eps_delta]
```

Step 2 again requires `_find_and_rewrite_all` because after step 1 the inner
`cross(b,c)` sits inside a `DoubleContract` argument, not at the root.

### Removal of asserted identities

- `epsilon.py`: `anti_commutativity` and `bac_cab` Identity objects removed;
  `ALL = []`.  The module docstring explains the promotion to theorems.
- `vectors.py`: `cross_anticomm` is now defined directly (no import from
  epsilon); it is added to `ALL` (5 items total).

## Why the proofs are non-circular

- `eps_anticomm` and `eps_delta` are statements about the ε *tensor* in index
  form; they do not mention cross products.
- `cross_def` is the *definition* `a×b := ε:(a⊗b)`, not derived from
  anti-commutativity.
- Together these suffice to derive both cross-product results without any
  circular dependency.
