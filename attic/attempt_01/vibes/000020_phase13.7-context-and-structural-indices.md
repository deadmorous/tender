# Phase 13.7 ‚Äî Context refactoring and structural (anonymous) index IDs

## Motivation

The current abstract-index notation (`SymBasisVec`, `IndexedSum`, the
`"a^i"`-style component symbols) uses letter strings as the identity of an
abstract index.  Two nodes are "paired" ‚Äî i.e., contracted ‚Äî if and only if
they carry the same letter.  This is fragile:

- Expanding two tensors in the same expression with the same default letter
  ("i", "j") produces four occurrences of each letter, breaking Einstein
  summation semantics.
- `simplify_basis_dot_step` infers pairing by parsing symbol strings and
  comparing letter characters ‚Äî an implicit convention rather than a
  structural property.
- Equality / normalization of `IndexedSum` nodes is letter-invariant only
  through a special-cased string comparison in `_normalize_component_form`.

In proper Einstein notation a **dummy index** has no intrinsic name.  Its name
is chosen at *render time* solely for human readability.  Phase 13.7 moves to
that model: index identity is an opaque integer **index ID**; letter names are
assigned when producing LaTeX and have no semantic role inside the AST.

---

## Pre-requisite: `tender::Context`

Every `make_*` factory function currently takes `ResourceList&`.  Adding
index-ID allocation would require touching every signature again.  Instead,
introduce a single `Context` object that is the one argument passed everywhere.
Future extensions (ID factories for other purposes, diagnostic metadata, etc.)
add fields to `Context` without further signature churn.

### New header: `src/include/tender/context.hpp`

```cpp
namespace tender {

struct Context {
    ResourceList rl;

    // Allocate a fresh, globally-unique index ID.
    // Used by Phase 13.7 to create anonymous abstract indices.
    auto alloc_index_id() -> int { return next_index_id_++; }

private:
    int next_index_id_ = 0;
};

} // namespace tender
```

`Context` owns its `ResourceList` by value.  Existing code that constructs a
`ResourceList` locally changes to `Context ctx; ... ctx.rl.make<...>(...)`.

### Mechanical rename

Replace every occurrence of `ResourceList& rl` with `Context& ctx` in:

- all `make_*` function declarations in `expr.hpp`, `coord_system.hpp`,
  `basis.hpp`, and any other public headers;
- all `make_*` definitions in the corresponding `.cpp` files;
- all test helpers (`make_rl()` ‚Üí `make_ctx()`, `auto rl = ‚Ä¶` ‚Üí `auto ctx = ‚Ä¶`);
- the Python binding (`g_rl` ‚Üí `g_ctx`, all lambda captures).

Within the implementations, `rl.make<T>(‚Ä¶)` becomes `ctx.rl.make<T>(‚Ä¶)`.

This is entirely mechanical and produces no semantic change.  It is a
prerequisite commit that lands before any Phase 13.7 work.

## Phase 13.7: structural index IDs

### Design

An **index ID** is a non-negative integer allocated by `ctx.alloc_index_id()`.
Two abstract-index nodes that share an index ID are contracted over that index.
No letter string is stored in the AST; letters appear only in rendered output.

#### Affected nodes

**`AbstractComp`** ‚Äî new node replacing the `NamedTensor("a^i", 0)` hack.

```cpp
class AbstractComp : public Expr {
public:
    // base_sym: the tensor's base name ("a", "A", "sigma", ‚Ä¶).
    // indices: list of (index_id, is_upper) pairs, one per abstract index.
    AbstractComp(std::string base_sym,
                 std::vector<std::pair<int,bool>> indices);

    [[nodiscard]] auto rank()    const noexcept -> int override { return 0; }
    [[nodiscard]] auto latex()   const -> std::string override;
    [[nodiscard]] auto python()  const -> std::string override;
    [[nodiscard]] auto base_sym() const -> std::string const& { return base_sym_; }
    [[nodiscard]] auto indices()  const
        -> std::vector<std::pair<int,bool>> const& { return indices_; }
};
```

`latex()` queries the active render context (see ¬ßRendering) for the letter
assigned to each index ID, then assembles `"A^{ij}_{k}"` style strings using
the same iterative `sym_to_latex` logic.

**`SymBasisVec`** ‚Äî `std::string letter_` ‚Üí `int index_id_`.

```cpp
class SymBasisVec : public Expr {
public:
    SymBasisVec(CoordSystem const& cs, int index_id, bool cobasis);
    [[nodiscard]] auto index_id()   const noexcept -> int  { return index_id_; }
    [[nodiscard]] auto is_cobasis() const noexcept -> bool { return cobasis_; }
    // latex() renders e.g. \mathbf{e}_{i} using the active render context.
};
```

**`IndexedSum`** ‚Äî `std::string index_letter_` ‚Üí `int index_id_`.

```cpp
class IndexedSum : public Expr {
public:
    IndexedSum(AbstractComp const* lhs, AbstractComp const* rhs, int index_id);
    [[nodiscard]] auto index_id() const noexcept -> int { return index_id_; }
    // latex() renders e.g. "a^{i} b_{i}" via render context.
};
```

#### `Context::alloc_index_id()`

Each call to `expand_in_basis_step` (abstract mode) allocates one fresh index
ID per tensor index being expanded:

```python
# rank-1
id0 = ctx.alloc_index_id()        # e.g. 7
comp = AbstractComp("a", [(7, True)])   # a^{i} (upper)
sbv  = SymBasisVec(cs, 7, cobasis=False)  # e_{i}
result = TensorProduct(comp, sbv)

# rank-2, covariant=[True, False]
id0, id1 = ctx.alloc_index_id(), ctx.alloc_index_id()  # e.g. 8, 9
comp = AbstractComp("A", [(8, True), (9, False)])  # A^{i}_{j}
result = TensorProduct(TensorProduct(comp,
             SymBasisVec(cs, 8, False)),   # e_{i}
             SymBasisVec(cs, 9, True))     # e^{j}
```

Because IDs are globally unique within a session, two independent expansions
never share an ID regardless of rank or covariant pattern.  The `letter`
parameter of `expand_in_basis_step` is retired for abstract mode.

#### `simplify_basis_dot_step` update

The current implementation parses `nt_a->symbol()` to extract a separator
character.  With `AbstractComp`, matching is structural:

```cpp
// Contract(TP(AbstractComp_a, SBV_basis), TP(AbstractComp_b, SBV_cobasis))
// ‚Üí IndexedSum if SBV_basis.index_id == SBV_cobasis.index_id
//              and they belong to the same CoordSystem
//              and AbstractComp_a / _b each have exactly that one index.
if (sbv_l.index_id() == sbv_r.index_id()
    && &sbv_l.cs() == &sbv_r.cs()
    && sbv_l.is_cobasis() != sbv_r.is_cobasis())
{
    return make_indexed_sum(ctx, comp_a, comp_b, sbv_l.index_id());
}
```

No string parsing, no separator convention.

### Rendering

`AbstractComp::latex()`, `SymBasisVec::latex()`, and `IndexedSum::latex()`
need to convert an index ID to a letter.  The mechanism:

**Approach: two-pass top-level renderer.**

Add a free function `render_latex(Expr const*)` that:

1. Walks the expression tree, collecting all index IDs in first-appearance
   (left-to-right, depth-first) order.
2. Assigns letters from the sequence `i, j, k, l, m, n, p, q, r, s`
   (extending with `a, b, c, ‚Ä¶` if needed).
3. Stores the `{id ‚Üí letter}` map in a thread-local `RenderCtx`.
4. Calls `expr->latex()` ‚Äî individual `latex()` implementations query the
   thread-local `RenderCtx` to resolve their ID.

The existing `Expr::latex()` virtual interface is unchanged.  The thread-local
`RenderCtx` is a thin wrapper:

```cpp
struct RenderCtx {
    std::unordered_map<int,std::string> id_to_letter;
    static RenderCtx const& current(); // reads thread-local
};
```

`latex()` on abstract-index nodes calls `RenderCtx::current().id_to_letter.at(index_id_)`.
If called outside a `render_latex` invocation (e.g. in a unit test that calls
`.latex()` directly), the map is empty and the node falls back to emitting the
ID as a subscript (e.g. `i_{7}`) ‚Äî visually odd but never silent-wrong.

`render_latex` is the only rendering entry point exposed through the Python
binding.  The existing `expr.latex()` Python property stays for backward
compatibility on concrete nodes; for abstract-index nodes it now calls
`render_latex` internally.

### Equality and normalization

`_normalize_component_form` currently special-cases `IndexedSum` by comparing
`(sym, sep)` pairs letter-invariantly.  With index IDs, normalization is:

1. Walk the expression; collect index IDs in depth-first order.
2. Rename IDs to `0, 1, 2, ‚Ä¶` (canonical order).
3. Serialize the renamed structure to a string.

Two `IndexedSum` nodes (or any two abstract-index expressions) are equal iff
their canonical serializations match.  This is structurally correct and handles
multi-index expressions automatically.

## API changes summary

| Old | New |
|-----|-----|
| `make_*(ResourceList& rl, ‚Ä¶)` | `make_*(Context& ctx, ‚Ä¶)` |
| `NamedTensor("a^i", 0)` (abstract component) | `AbstractComp("a", [(id, true)])` |
| `SymBasisVec(cs, "i", cobasis)` | `SymBasisVec(cs, id, cobasis)` |
| `IndexedSum(‚Ä¶, "i")` | `IndexedSum(comp_a, comp_b, id)` |
| `make_sym_basis_vec(cs, letter, cobasis)` | `make_sym_basis_vec(ctx, cs, id, cobasis)` |
| `expand_in_basis_step(‚Ä¶, letter="i")` | `expand_in_basis_step(‚Ä¶)` ‚Äî letter retired |
| `expr.latex()` | `render_latex(expr)` for abstract-index exprs |
| `_normalize_component_form` (letter-invariant string) | canonical ID serialization |
