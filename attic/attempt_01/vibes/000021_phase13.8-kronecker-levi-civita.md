# Phase 13.8 — Kronecker delta, Levi-Civita symbol, and basis expansions

## Motivation

Phase 13.7 established opaque integer index IDs and made every abstract-index
node structurally correct.  One loose end was deliberately left: when two
tensors are expanded independently — each via its own `expand_in_basis_step`
call — they receive different index IDs, so `simplify_basis_dot_step` never
fires (it requires both `SymBasisVec` nodes to share the same ID).

The mathematically correct fix is not a shortcut but a derivation step:
contraction of a basis vector with a cobasis vector yields a **Kronecker
delta**, and the Kronecker delta then contracts the dummy index.  Phase 13.8
introduces this derivation, plus the companion `LeviCivitaSymbol` node and the
standard basis expansions of `IdentityTensor` and `LeviCivitaTensor`.

Phase 13.8 also introduces the **`Theorem` type** — a `Theorem` is a proved
statement, verified within the system, as opposed to an `Identity` which is
an axiom or definition asserted without proof.  Three classical vector theorems
are proved as the self-test for this phase.

---

## Pre-requisite: `Theorem` type

### Motivation

`Identity` records a mathematical statement but does not require a proof: the
caller asserts it.  This is correct for axioms and definitions.  For derived
results — statements that follow from earlier ones — the system should *verify*
the claim before accepting it.  A `Theorem` is a proved `Identity`.

### Design

`Theorem` is a Python-level wrapper.  It holds the statement (as a pair of
`Expr` objects) plus a *proof callable* that is invoked at construction.  If
the callable raises or returns `False`, construction fails.

```python
class Theorem:
    """A mathematical statement that has been verified within the system."""

    def __init__(self, name: str, lhs: Expr, rhs: Expr, proof: callable):
        if not proof(lhs, rhs):
            raise ValueError(f"Theorem '{name}': proof failed")
        self.name = name
        self.lhs  = lhs
        self.rhs  = rhs

    # Convenience constructors -----------------------------------------------

    @classmethod
    def by_components(cls, name, lhs, rhs, **vars):
        """Prove lhs == rhs by reducing both sides to the same component form.
        
        Uses the existing prove_equal_by_components facility.  This works for
        any equational identity over WCS components.
        """
        return cls(name, lhs, rhs,
                   proof=lambda l, r: prove_equal_by_components(l, r, **vars))

    @classmethod
    def by_derivation(cls, name, lhs, rhs, steps: list, **vars):
        """Prove lhs == rhs by running a derivation from lhs and comparing to rhs
        via component equality.
        
        steps: list of DerivationStep that transform lhs toward rhs.
        """
        def proof(l, r):
            result = Derivation(steps).apply(State(l))[-1].expr()
            return prove_equal_by_components(result, r, **vars)
        return cls(name, lhs, rhs, proof=proof)
```

`Theorem` is usable anywhere `Identity` is accepted (it records `.lhs` and
`.rhs`), but it cannot be used as a rewrite rule via `apply_identity` until a
matching strategy is implemented for abstract-index theorems.

### Theorem library

A new module `tender.lib.theorems` stores proved theorems alongside
`tender.lib.identities`.  Theorems are instantiated at import time; a failed
proof fails the import.  This is the "self-test" property: if any proof
regresses, the library is immediately unloadable.

## Rendering: mixed-index dot-placeholder notation

### The rule

For abstract-index nodes (`AbstractComp`, `LeviCivitaSymbol`) whose indices are
a mix of upper and lower, the renderer writes **both** a full upper row and a
full lower row, using a centred dot `\cdot` as a placeholder in each row for
slots that belong to the other level.

```
rank-2, slots (upper, lower):
    upper row:  i  ·        lower row:  ·  j
    → A^{i{\cdot}}_{\cdot j}

rank-3, slots (upper, lower, lower):
    upper row:  i  ·  ·     lower row:  ·  j  k
    → \varepsilon^{i{\cdot\cdot}}_{\cdot jk}

rank-3, slots (upper, lower, upper):
    upper row:  i  ·  k     lower row:  ·  j  ·
    → \varepsilon^{i{\cdot}k}_{{\cdot}j{\cdot}}

rank-3, slots (lower, upper, lower):
    upper row:  ·  j  ·     lower row:  i  ·  k
    → \varepsilon^{{\cdot}j{\cdot}}_{i{\cdot}k}
```

When all indices are at the same level, no dots are needed:
```
all lower  →  \varepsilon_{ijk}
all upper  →  \varepsilon^{ijk}
```

The upper group is always written first: `\base^{…}_{…}`.

### Scope

This dot-placeholder rendering applies to `AbstractComp` and `LeviCivitaSymbol`.
It does **not** apply to `KroneckerDelta`: `\delta_{i}^{j}` has a fixed
two-index convention that is universally understood, so no dots are needed
there.

### Phase 13.7 rendering limitation fixed here

Phase 13.7's `AbstractComp::latex` rendered alternating-level indices by
emitting multiple separate script groups (e.g. `A^{i}_{j}^{k}`), which is
awkward to read.  The dot-placeholder scheme replaces this: there is exactly
one upper group and one lower group per node, giving a clean two-row layout.

## New AST nodes

### `KroneckerDelta`

A rank-0 node carrying two index IDs and their upper/lower status.  The
canonical form is `δ_{lower}^{upper}` (one lower, one upper), but the node
can represent any combination since the upper/lower information is stored
explicitly.

```cpp
class KroneckerDelta : public Expr {
public:
    KroneckerDelta(int lower_id, int upper_id);

    [[nodiscard]] auto rank()     const noexcept -> int  override { return 0; }
    [[nodiscard]] auto lower_id() const noexcept -> int  { return lower_id_; }
    [[nodiscard]] auto upper_id() const noexcept -> int  { return upper_id_; }
    [[nodiscard]] auto latex(IndexNameMap const&) const -> std::string override;
    [[nodiscard]] auto python()                   const -> std::string override;

private:
    int lower_id_;
    int upper_id_;
};

auto make_kronecker_delta(ResourceList& rl, int lower_id, int upper_id)
    -> KroneckerDelta*;
```

`latex(map)` renders as `\delta_{i}^{j}`.  No dot placeholder is used because
`KroneckerDelta` always has exactly two indices in fixed positions.

`collect_index_ids_impl` visits `lower_id_` then `upper_id_`.

### `LeviCivitaSymbol`

A rank-0 node carrying three index IDs with their upper/lower flags.  Distinct
from `LeviCivitaTensor` (the rank-3 invariant tensor): `LeviCivitaSymbol`
is the numerical component array `ε_{ijk}` or its mixed variants.

```cpp
class LeviCivitaSymbol : public Expr {
public:
    // ids[k] / upper[k] are the index ID and upper/lower flag for slot k.
    LeviCivitaSymbol(std::array<int,3> ids, std::array<bool,3> upper);

    [[nodiscard]] auto rank()  const noexcept -> int  override { return 0; }
    [[nodiscard]] auto ids()   const noexcept -> std::array<int,3> const&  { return ids_; }
    [[nodiscard]] auto upper() const noexcept -> std::array<bool,3> const& { return upper_; }
    [[nodiscard]] auto latex(IndexNameMap const&) const -> std::string override;
    [[nodiscard]] auto python()                   const -> std::string override;

private:
    std::array<int,3> ids_;
    std::array<bool,3> upper_;
};

auto make_levi_civita_symbol(
    ResourceList& rl,
    std::array<int,3> ids,
    std::array<bool,3> upper) -> LeviCivitaSymbol*;
```

`latex(map)` uses the dot-placeholder scheme (see §Rendering):

| `upper` | rendering |
|---------|-----------|
| `{false, false, false}` | `\varepsilon_{ijk}` |
| `{true,  true,  true}`  | `\varepsilon^{ijk}` |
| `{true,  false, false}` | `\varepsilon^{i{\cdot\cdot}}{}_{\cdot jk}` |
| `{false, false, true}`  | `\varepsilon^{{\cdot\cdot}k}{}_{ij{\cdot}}` |
| `{true,  false, true}`  | `\varepsilon^{i{\cdot}k}{}_{{\cdot}j{\cdot}}` |
| `{false, true,  false}` | `\varepsilon^{{\cdot}j{\cdot}}{}_{i{\cdot}k}` |

Every slot's ordinal position is unambiguous: if you see `i` in position 2
of the upper row, index `i` is in slot 2 and is contravariant.

`collect_index_ids_impl` visits `ids_[0]`, `ids_[1]`, `ids_[2]` in order.

## Simplification rules

### Rule 1: basis × cobasis → Kronecker delta

Generalise `simplify_basis_dot_step` to handle the case where the two
`SymBasisVec` nodes have *different* index IDs.

Current behaviour (same ID, shortcut):
```
Contract(TP(AC_a(i), SBV(i, basis)), TP(AC_b(i), SBV(i, cobasis)))
  → AbstractIndexedSum(AC_a, AC_b, i)
```

New behaviour (different IDs, general):
```
Contract(TP(AC_a(i), SBV(i, basis)), TP(AC_b(j), SBV(j, cobasis)))   i ≠ j
  → Product(Product(AC_a, AC_b), KroneckerDelta(lower=i, upper=j))
```

The rendering of the intermediate result is `a^{i} b_{j} \delta_{i}^{j}`.
`contract_kronecker_step` (Rule 3 below) then collapses this to `a^{i} b_{i}`.

The existing same-ID shortcut remains valid and is kept for the case where the
user constructs both sides with the same ID manually.

### Rule 2: Kronecker delta with equal IDs → 1

```
KroneckerDelta(id, id)  →  RationalConst(1)
```

Applied as a constant fold inside `make_kronecker_delta`.

### Rule 3: dummy-index contraction (`contract_kronecker_step`)

A new C++ `DerivationStep` factory — not an `Identity`.  It requires a global
pass over the expression to count index-ID occurrences.

Algorithm:

1. Walk the full expression tree; for each `KroneckerDelta(lo, hi)` found,
   count total occurrences of `lo` and `hi` across the entire tree.
2. Identify the *dummy* index: the one with count == 2 (appears once inside
   the delta, once elsewhere).  If both have count 2, choose `hi` as dummy
   (replace `hi` with `lo`).  If neither has count 2, leave the delta in place.
3. For dummy `d` being replaced by free `f`:
   a. Rebuild the entire tree with `d` replaced by `f` in every abstract-index
      node, using a new recursive helper
      `substitute_index(Expr*, int old_id, int new_id) -> Expr*`.
   b. Replace the `KroneckerDelta` node with `RationalConst(1)`.

Affected node types for `substitute_index`: `AbstractComp`, `SymBasisVec`,
`AbstractIndexedSum`, `KroneckerDelta`, `LeviCivitaSymbol`.

`contract_kronecker_step` contracts one delta per application.  Multiple
deltas are handled by applying the step repeatedly.

### Rule 4: Kronecker delta with concrete integer indices → 0 or 1

```
KroneckerDelta(lo, hi)  →  0  if lo ≠ hi,   1  if lo == hi
   (where lo and hi resolve to concrete integers in [1, dim])
```

Relevant when Kronecker deltas survive into a partially-concrete expression.
Deferred to Phase 13.9 if not immediately needed.

## Basis expansions

### Identity tensor

**Cobasis–basis dyad (orthonormal):**
```
I  →  e^{i} ⊗ e_{i}
   =  SBV(id, cobasis) ⊗ SBV(id, basis)       [single shared index ID]
```

**Explicit Kronecker-delta form (general, two IDs):**
```
I  →  δ_{i}^{j} e^{i} ⊗ e_{j}
   =  KroneckerDelta(i, j) ⊗ SBV(i, cobasis) ⊗ SBV(j, basis)
```
After `contract_kronecker_step` this reduces to the first form.

Implemented as `expand_identity_step(cs)`.  Raises `NotImplementedError` for
non-orthonormal CS (metric tensor support is deferred).

### Levi-Civita tensor

```
eps  →  ε_{ijk} e^{i} ⊗ e^{j} ⊗ e^{k}
     =  LeviCivitaSymbol([i,j,k], all_lower)
        ⊗ SBV(i, cobasis) ⊗ SBV(j, cobasis) ⊗ SBV(k, cobasis)
```

Three fresh index IDs.  Implemented as `expand_levi_civita_step(cs)`.

Restricted to orthonormal CS.

## Identities library additions

These are *asserted* statements (axioms at this level of the system).  They
do not require index-aware pattern matching to *state*; that is only needed
to *apply* them as rewrite rules.

| Name | Statement |
|---|---|
| `kronecker_trace` | `δ_{i}^{i} = n`  (trace = dimension; n = cs.dim) |
| `kronecker_contract` | `δ_{i}^{j} a^{i} = a^{j}` — handled by `contract_kronecker_step`; no rewrite rule entry needed |
| `lc_self_dot_3d` | `ε_{ijk} ε^{ijk} = 6` |
| `lc_one_shared` | `ε_{ijk} ε^{iml} = δ_{j}^{m} δ_{k}^{l} - δ_{j}^{l} δ_{k}^{m}` |
| `lc_two_shared` | `ε_{ijk} ε^{ijl} = 2 δ_{k}^{l}` |
| `cross_via_lc` | `(a × b)_{k} = ε_{ijk} a^{i} b^{j}` |

Phase 13.8 delivers `kronecker_trace` as a usable identity (its LHS is a
scalar rank-0 node, matchable structurally).  The Levi-Civita identities are
stored as stubs — statements recorded for Phase 13.9, which will add
index-aware matching.

## Theorems: three classical vector identities

These become the Phase 13.8 self-test.  All three are proved by
`Theorem.by_components` at import time.  They live in
`tender.lib.theorems`.

### `dot_commutativity`
**Statement**: `a · b = b · a`

```python
a, b = tensor("a", 1), tensor("b", 1)
dot_commutativity = Theorem.by_components(
    "dot_commutativity",
    lhs = a @ b,
    rhs = b @ a,
    a = wcs, b = wcs,
)
```

### `cross_anticommutativity`
**Statement**: `a × b = -(b × a)`

```python
cross_anticommutativity = Theorem.by_components(
    "cross_anticommutativity",
    lhs = a % b,
    rhs = -(b % a),
    a = wcs, b = wcs,
)
```

### `bac_cab`
**Statement**: `a × (b × c) = b(a · c) - c(a · b)`

```python
a, b, c = tensor("a", 1), tensor("b", 1), tensor("c", 1)
bac_cab = Theorem.by_components(
    "bac_cab",
    lhs = a % (b % c),
    rhs = b * (a @ c) - c * (a @ b),
    a = wcs, b = wcs, c = wcs,
)
```

### Note on derivation proofs

The `by_components` proofs above are *verification* proofs — they confirm
correctness by evaluating components.  Full algebraic derivation proofs of
these theorems (step-by-step index manipulations ending at the other side)
require the Levi-Civita identity `ε_{ijk} ε^{iml} = …` and are deferred to
Phase 13.9, which will add `Theorem.by_derivation` proofs that make the
algebraic reasoning explicit.

## Pattern matching challenge

Existing `Identity` + `apply_identity` matching is *structural* — it matches
subtree shapes via `PatternVar`.  It does not understand index IDs.

An identity like `ε_{ijk} ε^{iml} = δ_{j}^{m} δ_{k}^{l} - δ_{j}^{l} δ_{k}^{m}`
requires knowing that one index (i) appears in both `LeviCivitaSymbol` nodes
while j, k, m, l are free — a global property of the index structure, not a
subtree shape.

Three design options (see Phase 13.7 vibe discussion, §Pattern matching):

**Option A — `IndexPatternVar`**: extend `PatternVar` with index-ID matching.  
**Option B — `IndexIdentity`**: a separate identity type with its own engine.  
**Option C — hard-coded step factories**: each rule is its own function.

Phase 13.8 uses **Option C** for `contract_kronecker_step`.  The choice
between A and B for the Levi-Civita identities is deferred to Phase 13.9.

## Exit criterion

**Theorems self-test**: `import tender` succeeds, meaning all three theorems in
`tender.lib.theorems` pass their component proofs at import time.

**Full Kronecker path**: the following derivation produces correct, human-
readable LaTeX at every step with no index letter appearing more than twice:

```python
a, b = tensor("a", 1), tensor("b", 1)
expr = a @ b
steps = [
    expand_in_basis_step(a, wcs, covariant=True,  abstract=True),
    expand_in_basis_step(b, wcs, covariant=False, abstract=True),
    simplify_basis_dot_step(wcs),   # → a^{i} b_{j} \delta_{i}^{j}
    contract_kronecker_step(),      # → a^{i} b_{i}
]
Derivation(steps).apply(State(expr))
```

**Rendering**: `LeviCivitaSymbol` with mixed upper/lower indices (e.g. upper =
`{True, False, True}`) renders as `\varepsilon^{i}{}_{j}{}^{k}` — slot order
preserved, `{}` separators present, no ambiguity.
